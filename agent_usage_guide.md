# 안티그래비티 에이전트 활용 가이드 (SOP)

이 가이드는 `_agent` 폴더에 생성된 워크플로우와 스킬을 사용하여 프로젝트의 품질을 유지하고 효율적으로 협업하는 방법을 설명합니다.

## 1. 세션 시작 (맥락 파악)
새로운 대화를 시작하거나 오랜만에 작업을 재개할 때 사용합니다.
- **워크플로우**: `load-recent-changes`
- **방법**: `/load-recent-changes` 실행 요청
- **효과**: 최근 어떤 파일이 수정되었고 어떤 커밋이 있었는지 제가 빠르게 파악하여 별도의 긴 설명 없이 작업을 이어갈 수 있습니다.

## 2. 개발 중 (중간 저장)
큰 기능을 구현하는 도중 잠시 멈추거나, 다른 작업을 위해 맥락을 보존해야 할 때 사용합니다.
- **워크플로우**: `commit-session`
- **방법**: `/commit-session` 실행 요청
- **효과**: 중간 작업물을 `WIP` 커밋으로 안전하게 저장합니다. 코드를 날릴 걱정 없이 안심하고 대화를 종료할 수 있습니다.

## 3. 기능 구현 완료 후 (품질 검증)
코딩을 마친 후 저에게 리뷰를 요청하기 직전에 사용합니다.
- **스킬**: `verify-implementation`
- **방법**: `/verify-implementation` 실행 요청
- **효과**: 프로젝트의 모든 규칙(한글 주석, SOLID 원칙, 파일 구조 등)을 자동으로 체크합니다. 실수를 미리 잡아내어 코드 품질을 높입니다.

## 4. 최종 병합 및 정리 (마무리)
특정 작업(예: 신규 기능 추가)이 완전히 끝나서 `main` 브랜치나 다른 베이스 브랜치에 반영할 때 사용합니다.
- **스킬**: `merge-worktree`
- **방법**: `/merge-worktree` 실행 요청
- **효과**: 지저분한 `WIP` 커밋들을 하나로 합쳐서(Squash) 아주 깔끔하고 전문적인 커밋 메시지와 함께 병합합니다. 프로젝트 히스토리가 훨씬 보기 좋아집니다.

---

## 추천 시나리오
1. **시작**: "지지난번에 하던 작업 이어서 하고 싶은데, `/load-recent-changes` 실행해서 맥락 좀 봐줘."
2. **개발 중**: (코딩 후) "지금까지 한 거 `/commit-session`으로 세이브해줘."
3. **마무리**: "기능 다 만든 것 같아. `/verify-implementation` 돌려서 문제 없는지 확인하고, 통과하면 `/merge-worktree`로 병합해줘."
