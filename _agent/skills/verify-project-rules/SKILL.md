# `verify-project-rules` 스킬

이 스킬은 `shortsmaker` 프로젝트의 고유한 개발 규칙이 소스 코드에 잘 반영되어 있는지 자동으로 점검합니다.

## 1. 검증 대상 규칙

1.  **언어 규칙 (Rule 1)**: 모든 소스 코드의 주석 및 독스트링(Docstring)은 **한국어**로 작성되어야 합니다.
2.  **설계 규칙 (Rule 7)**:
    -   **SRP (단일 책임 원칙)**: 복잡한 로직은 별도의 함수로 분리되어야 합니다.
    -   **함수 복잡도**: 단일 함수가 너무 비대해지지 않도록 관리합니다. (권장: 50라인 이내)
3.  **한글 리소스 관리**: 프로젝트 문서(README.md, TASK.md 등)의 최신화 여부를 확인합니다.

## 2. 검증 절차 (Workflow)

에이전트는 다음 단계를 수행하여 보고서를 생성해야 합니다.

### 단계 1: 소스 코드 주석 언어 검사
- `grep` 또는 유사 도구를 사용하여 소스 코드 내의 주석(`#`, `"""`)을 추출합니다.
- 영어 비중이 너무 높거나 한글이 전혀 없는 주요 로직 파일을 찾아냅니다.
- **체크리스트**:
    - [ ] `src/shortsmaker/web/main.py`
    - [ ] `src/shortsmaker/renderer/ffmpeg.py`

### 단계 2: 함수 비대화(Bloated Function) 인지
- 주요 모듈 내에서 50라인 이상을 차지하는 함수가 있는지 확인합니다.
- 특히 `index` 함수나 `build_command` 류의 복잡한 로직을 집중 점검합니다.

### 단계 3: 문서 연동성 확인
- `TASK.md`에 진행 중이거나 완료된 작업이 비어있지 않은지 확인합니다.

## 3. 결과 보고 양식

검증 완료 후 다음과 같은 형식으로 사용자에게 보고합니다.

```markdown
### 🔍 프로젝트 규칙 검증 결과

- **언어 규칙**: [✅ 통과 / ⚠️ 보강 필요] (이유 설명)
- **설계 규칙**: [✅ 통과 / ⚠️ 리팩토링 권장] (대상 함수 명시)
- **문서화**: [✅ 통과 / ⚠️ 업데이트 필요]
```
