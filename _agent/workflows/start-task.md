# 새 작업 시작 (Start Task)

이 워크플로우는 새로운 기능을 개발하거나 버그를 수정하기 위해 깨끗한 작업 환경(Git Worktree)을 구성합니다.

## 실행 절차 (Instructions)

사용자가 새 작업의 이름(브랜치명)을 제공하면 다음 단계를 순차적으로 수행합니다.

### 단계 1: 준비 및 검증
1. **메인 브랜치 상태 확인**: **중요**: 현재 메인 폴더(`main` 브랜치)에 커밋되지 않은 변경 사항이 있는지 확인합니다. 메인 브랜치가 깨끗하지 않으면 경로 혼동으로 인한 데이터 유실 위험이 있습니다.
2. **타겟 브랜치 확인**: 기본적으로 `main` 브랜치를 기준으로 합니다.
2. **브랜치 이름 유효성 검사**: 특수문자나 공백이 없는지 확인하고 필요시 케밥 케이스(kebab-case)로 변환합니다.
3. **중복 확인**: 동일한 이름의 브랜치나 워크트리 폴더가 이미 존재하는지 확인합니다.

### 단계 2: 브랜치 및 워크트리 생성
1. **브랜치 생성**: `git branch <branch-name> main` 실행.
2. **워크트리 경로 결정**: 프로젝트 내부의 `.worktrees/` 디렉토리에 `<branch-name>` 이름으로 경로를 설정합니다. (예: `.worktrees/feat-new-room`)
3. **워크트리 추가**: `git worktree add .worktrees/<branch-name> <branch-name>` 실행.

### 단계 3: 환경 구성
1. **패키지 설치**: 새 워크트리 폴더로 이동하여 `npm install`을 실행합니다. (필요 시)
2. **초기화 확인**: 프로젝트의 기본 빌드나 린트가 통과하는지 가볍게 체크합니다.

### 단계 4: 작업 진행 및 보고
1. **작업 수행**: 워크트리 내부에서 구현 및 테스트를 진행합니다.
2. **개발 완료 보고**: 모든 작업이 완료되면 `git diff`를 통해 변경 사항을 사용자에게 보고하고 **머지 승인을 요청**합니다.
3. **병합 (사용자 승인 시에만)**: 사용자가 머지를 승인하면 `merge-worktree` 스킬에 따라 병합을 진행합니다. **중요: 사용자의 명시적 요청 없이 머지를 자동 수행하지 않습니다.**
